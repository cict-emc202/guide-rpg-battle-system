Introduction

In this guide, we will build a turn-based RPG battle system from scratch in Godot 4.x. You'll learn how to set up a battle scene, integrate animated sprite sheets, manage health bars, wire up UI signals, and implement attack/heal/defend mechanics — all explained step by step for beginners.

What we're building:

Our goal: a battle scene with animated characters, health bars, and command buttons.

3 Scene Setup

Now let's build the scene step by step.

3.1 — Create the Root Scene

  1. Open Godot and create a New Scene.
  2. Choose Node2D as your root node.
  3. Rename it to BattleScene.
  4. Save the scene as battle_scene.tscn.

Setting up the scene.

3.2 — Add the Background

  1. Add a Sprite2D child node to BattleScene.
  2. Rename it Background.
  3. Drag backgrounds/1.png into the Texture property in the Inspector.
  4. Set Offset → Centered to Off so the image starts from the top-left corner.

3.3 — Add the Hero & Monster

We use AnimatedSprite2D instead of plain Sprite2D so we can play idle, attack, and hit animations.

  1. Add an AnimatedSprite2D child under BattleScene. Rename it Hero.
  2. Add another AnimatedSprite2D. Rename it Monster.

3.4 — Add Health Bars

  1. Select the Hero node. Add a TextureProgressBar child. Rename it HeroHealthBar.
  2. In the Inspector, set these properties:
    • Min Value: 0
    • Max Value: 100
    • Value: 100
    • Texture Under: drag HUD/bar/background.png
    • Texture Progress: drag HUD/bar/green.png
  3. Repeat for the Monster node, but use HUD/bar/red.png for the progress texture.

3.5 — Add the UI Layer

  1. Add a CanvasLayer child under BattleScene. Rename it UI.
  2. Under UI, add a Label named LogLabel — this will show battle messages.
  3. Add an HBoxContainer named ButtonContainer and position it at the bottom of the screen.
  4. Inside ButtonContainer, add three Button nodes: AttackButton, HealButton, and DefendButton.
  5. Set each button's Text property: ⚔️ Attack, 💊 Heal, 🛡️ Defend.

4 Setting Up Animated Sprites

This is where your characters come to life! Instead of static images, we'll use sprite sheets — a grid of animation frames in a single image.

📖 What is a Sprite Sheet? A sprite sheet packs multiple frames of animation into one image file. The engine reads the sheet by splitting it into equal-sized rectangles (frames), then displays them in sequence to create animation.

4.1 — Create a SpriteFrames Resource

  1. Select the Hero (AnimatedSprite2D) node.
  2. In the Inspector, find the Sprite Frames property and click <empty> → New SpriteFrames.
  3. Click the SpriteFrames resource to open the SpriteFrames Editor at the bottom of the screen.

4.2 — Add the Idle Animation

  1. In the Animations panel (left side), you'll see a "default" animation. Rename it to idle.
  2. Click the grid icon (📐 "Add frames from Sprite Sheet") button.
  3. Select char/paladin/sprite-sheet-249x100.png.
  4. A grid dialog will appear. Set:
    • Horizontal: the number of columns in the sheet
    • Vertical: the number of rows in the sheet
  5. Select only the idle frames (typically the first row) and click Add.
  6. Set the animation FPS to 8 for a natural pace.
  7. Check the Loop toggle ✅ — idle should repeat continuously.

4.3 — Add Attack & Hit Animations

  1. Click the + Add Animation button to create a new animation named attack.
  2. Repeat the sprite sheet import process, selecting the attack frames from the sheet.
  3. Set FPS to 10. Uncheck Loop — attack should play only once.
  4. Create another animation named hit, import the hit frames, set FPS to 8, no loop.

4.4 — Playing Animations in Code

Here's how to trigger animations from your script:

animation_logic.gd 
func play_hero_attack():
    # Switch to the attack animation
    hero.play("attack")
    
    # Wait for the animation to finish
    await hero.animation_finished
    
    # Return to idle
    hero.play("idle")

func play_hero_hit():
    hero.play("hit")
    await hero.animation_finished
    hero.play("idle")
💡 Tip: The animation_finished signal is emitted when a non-looping animation reaches its last frame. We await it so the code pauses until the animation is done, then we switch back to idle.

4.5 — Set Up the Monster

Repeat Steps 4.1–4.3 for the Monster node. If the monster doesn't have a sprite sheet, you can use the static monster/giant.png and create a simple "shake" effect using a Tween instead:

juice_effects.gd 
func shake_monster():
    var original_pos = monster.position
    var tween = create_tween()
    tween.tween_property(monster, "position", 
        original_pos + Vector2(10, 0), 0.05)
    tween.tween_property(monster, "position", 
        original_pos - Vector2(10, 0), 0.05)
    tween.tween_property(monster, "position", 
        original_pos, 0.05)
    await tween.finished

5 Connecting Signals

Signals are Godot's way of connecting events (like a button press) to functions in your script. This is a crucial concept for beginners!

📖 What are Signals? Think of signals as "announcements." When a button is pressed, it emits a signal — like shouting "I was pressed!" Your script can connect to that signal and respond with a function.

Choose the method that works best for you:

👍 Recommended for beginners. The editor makes it easy to visually connect signals without typing code.
  1. Select the AttackButton node in the Scene tree.
  2. Go to the Node tab (next to Inspector) on the right panel.
  3. Find pressed() in the signal list and double-click it.
  4. A dialog appears. Select BattleScene as the target node.
  5. The method name will auto-fill as _on_attack_button_pressed. Click Connect.
  6. Repeat for HealButton → _on_heal_button_pressed and DefendButton → _on_defend_button_pressed.
📖 How do I know it worked? After connecting, you'll see a green "connected" icon (🔗) next to the function in your script. If you click on the node, the Node tab will also show the connected signal with an arrow pointing to the target method.
💪 Preferred by experienced developers. Connecting signals in code gives you more control and keeps all your logic in one place.

Add these lines inside your _ready() function to connect each button's pressed signal to the corresponding handler function:

signals_setup.gd 
func _ready():
    # Connect button signals to their handler functions
    attack_btn.pressed.connect(_on_attack_button_pressed)
    heal_btn.pressed.connect(_on_heal_button_pressed)
    defend_btn.pressed.connect(_on_defend_button_pressed)

    # Initialize the battle
    hero.play("idle")
    update_ui()
    log_label.text = "A wild Giant appears!"
📖 How does .connect() work? Every signal in Godot is an object. Calling signal_name.connect(my_function) tells Godot: "When this signal fires, call my_function." The function name is passed without parentheses — you're giving a reference, not calling it.
⚠️ Common Error (both methods): If you see "Cannot connect signal: method not found", double-check that the function name in your script matches exactly — including underscores and capitalization.

7 The Battle Script

Now for the main event — the GDScript that powers our battle! Attach a new script to BattleScene. Let's build it piece by piece.

6.1 — Node References & Variables

First, we set up references to our scene nodes and declare our stats:

battle_scene.gd (Setup) 
@onready var hero = $Hero
@onready var monster = $Monster
@onready var hero_health_bar = $Hero/HeroHealthBar
@onready var monster_health_bar = $Monster/MonsterHealthBar
@onready var log_label = $UI/LogLabel
@onready var attack_btn = $UI/ButtonContainer/AttackButton
@onready var heal_btn = $UI/ButtonContainer/HealButton
@onready var defend_btn = $UI/ButtonContainer/DefendButton

var hero_hp : int = 100
var hero_max_hp : int = 100
var monster_hp : int = 100
var is_hero_turn : bool = true
var is_defending : bool = false
var battle_over : bool = false
📖 About @onready: The @onready annotation assigns the variable when the node enters the scene tree. Without it, the nodes might not exist yet and you'd get a null error. Always use @onready for node references.

6.2 — Initialization

battle_scene.gd (Lifecycle) 
func _ready():
    hero.play("idle")
    monster.play("idle")
    update_ui()
    log_label.text = "A wild Giant appears! Choose your action."

func update_ui():
    hero_health_bar.value = hero_hp
    monster_health_bar.value = monster_hp

func set_buttons_disabled(disabled: bool):
    attack_btn.disabled = disabled
    heal_btn.disabled = disabled
    defend_btn.disabled = disabled

6.5 — Attack Action

battle_scene.gd (Combat) 
func _on_attack_button_pressed():
    if not is_hero_turn or battle_over: return
    
    set_buttons_disabled(true)
    is_defending = false
    
    # 1. Play attack animation
    hero.play("attack")
    await hero.animation_finished
    hero.play("idle")
    
    # 2. Damage logic + Shake effect!
    var damage = randi_range(15, 25)
    monster_hp = max(0, monster_hp - damage)
    
    log_label.text = "Hero deals " + str(damage) + " damage!"
    update_ui()
    
    # Trigger the shake effect we wrote earlier
    await shake_monster()
    
    if check_battle_end(): return
    
    is_hero_turn = false
    await get_tree().create_timer(1.0).timeout
    monster_turn()

6.6 — Heal Action

battle_scene.gd (Heal) 
func _on_heal_button_pressed():
    if not is_hero_turn or battle_over: return
    
    set_buttons_disabled(true)
    is_defending = false
    
    var heal_amount = 25
    hero_hp = min(hero_max_hp, hero_hp + heal_amount)
    
    log_label.text = "Hero heals " + str(heal_amount) + " HP!"
    update_ui()
    
    is_hero_turn = false
    await get_tree().create_timer(1.0).timeout
    monster_turn()

6.7 — Monster's Turn

battle_scene.gd (Enemy AI) 
func monster_turn():
    if battle_over: return
    
    var base_damage = randi_range(8, 18)
    var actual_damage = base_damage
    
    if is_defending:
        actual_damage = base_damage / 2
        log_label.text = "Hero blocks! Takes only " + str(actual_damage) + " damage."
    else:
        log_label.text = "Monster hits Hero for " + str(actual_damage) + "!"
    
    hero.play("hit")
    await hero.animation_finished
    hero.play("idle")
    
    hero_hp = max(0, hero_hp - actual_damage)
    update_ui()
    
    if check_battle_end(): return
    
    is_hero_turn = true
    is_defending = false
    set_buttons_disabled(false)

8 Bonus: Defend Action

The Defend button adds tactical depth. When the hero defends, incoming damage is halved on the next monster turn.

battle_scene.gd (Defend) 
func _on_defend_button_pressed():
    if not is_hero_turn or battle_over: return
    
    set_buttons_disabled(true)
    is_defending = true
    log_label.text = "Hero braces for impact!"
    
    is_hero_turn = false
    await get_tree().create_timer(1.0).timeout
    monster_turn()
💡 Design Insight: The defend mechanic creates meaningful choices. Should the player heal (recover HP) or defend (reduce incoming damage)?

9 Victory & Defeat

Every battle needs an ending! Let's detect when either the hero or monster runs out of HP:

battle_scene.gd (Conclusion) 
func check_battle_end() -> bool:
    if monster_hp <= 0:
        log_label.text = "🎉 Victory! The Monster is defeated!"
        battle_over = true
        set_buttons_disabled(true)
        return true
    
    if hero_hp <= 0:
        log_label.text = "💀 Defeat... The Hero has fallen."
        battle_over = true
        set_buttons_disabled(true)
        return true
    
    return false
📖 Why return a bool? By returning true when the battle ends, we can use if check_battle_end(): return in our attack/heal functions to immediately stop processing. This prevents the monster from attacking after it's already dead!

📚 GDScript Glossary

Quick reference for the GDScript concepts used in this tutorial:

@onready

Assigns a variable when the node enters the scene tree. Used for node references so they're guaranteed to exist.

$NodePath

Shorthand for get_node("NodePath"). Finds a child node by its path relative to the current node.

await

Pauses the function until a signal is emitted. Used to wait for timers, animations, or other async operations.

signal

An event notification system. Nodes emit signals (like pressed) and other nodes can respond by connecting a function.

randi_range(min, max)

Returns a random integer between min and max (inclusive). Perfect for damage variation.

AnimatedSprite2D

A node that plays frame-based animations from a SpriteFrames resource. Supports multiple named animations (idle, attack, etc).

SpriteFrames

A resource that holds animation data — frame images, playback speed, and loop settings — for an AnimatedSprite2D.

TextureProgressBar

A progress bar that uses textures for its appearance. Set Under (background) and Progress (fill) textures.

Tween

Smoothly animates properties over time. Great for shake effects, fading, and movement without keyframe animations.

create_timer(sec)

Creates a one-shot timer. Combined with await, it pauses execution for the specified seconds.

❓ Mini Quiz

Test what you've learned! This quiz is not graded — it's just for self-assessment. Pick the best answer for each question, then check your score.

1

What is the correct node type for displaying animated character sprites?

AnimatedSprite2D is specifically designed for frame-based sprite animations using a SpriteFrames resource.
2

What does the @onready annotation do?

@onready delays assignment until the node is in the scene tree, ensuring referenced nodes exist.
3

How do you connect a button's pressed signal via GDScript?

In Godot 4.x, signals use the .connect() method directly: signal.connect(callable).
4

What does randi_range(10, 25) return?

randi_range(min, max) returns a random integer between min and max, both inclusive.
5

Which node is used for the health bar in this tutorial?

TextureProgressBar lets you use custom textures for the bar's background and fill, matching the pixel art style.
6

Why do we use await hero.animation_finished after playing an attack?

await pauses execution of the current function until the specified signal is emitted — in this case, when the animation reaches its last frame.
7

What does the Defend action do in our battle system?

When is_defending = true, the monster's damage is divided by 2 before being applied to the hero's HP.
8

What is a SpriteFrames resource?

SpriteFrames stores named animations, each with frame images, FPS, and loop settings, used by AnimatedSprite2D.
9

What does create_tween() do?

Tween smoothly interpolates properties (position, opacity, scale, etc.) from one value to another over a set duration.
10

Why does check_battle_end() return a bool?

Returning true lets us write if check_battle_end(): return — stopping the function early so the monster doesn't attack after dying.
